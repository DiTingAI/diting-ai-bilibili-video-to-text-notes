Thanks, Andrew. What I've seen over the last couple of years is that AI-assisted coding has evolved rapidly. It started from people asking OMS occasional coding questions to GitHub autocomplete, and then various tools that became more and more autonomous. Cloud Code, when it was released, was definitely a step up in terms of the degree of agency or the amount of tasks that the coding assistant could do by itself.

I think many people were surprised that you could set a task that Cloud Code would work on for many minutes or sometimes even more than a few minutes. Now there are developers orchestrating not just a single code instance, but even several of them working in parallel on different parts of a codebase. But coordinating all this and mastering best practices is not widely known. If you have not worked with people close to best practices, I think there could be a big uplift in mastering these practices and understanding how they drive that amazing productivity that many developers achieve using Cloud Code.

As we start to talk about those best practices, a key tip for working with Cloud Code is providing clear context to help Cloud Code achieve the task efficiently. This means pointing Cloud Code to the relevant files, clearly describing the features and functionality that you want, and making sure that you're properly extending the capabilities of Cloud Code with MCP servers and other tools in its ecosystem.

In this course, you'll apply those best practices to three different examples. We'll start with a rag chatbot, implementing features from the front end to the back end, including refactoring code, writing tests, and using the GitHub integration to work with pull requests and fix issues. You'll make use of many Cloud Code features like planning, thinking modes, creating parallel sessions, and managing Cloud's memory.

For the second example, we'll shift gears and work with Jupyter Notebooks to explore e-commerce data. We'll refactor notebooks using Cloud Code, remove redundant code, and create powerful dashboards with web applications.

Finally, we'll move to create a visual mockup based in Figma and use Cloud Code, the Figma MCP server, and a different MCP server to import the design, iterate, test, and build agentically a powerful front-end application.

If you're not currently a Cloud Code user, I think learning this set of ideas will give you a meaningful acceleration in the way you engineer systems. Even if you are a current Cloud Code user, having Ellie share these best practices with you in a comprehensive and systematic way will, I hope, leave you with quite a few new things to try that will be useful for your work.

I'd like to thank Harah Salami from D-Learn.ai, who contributed to this course. In the next video, Ellie will share Cloud Code's underlying architecture, and you might be surprised by how simple the architecture is. Cloud Code relies on just a small number of tools to search for patterns within your code files, list directories, look at files, and execute reggags.

It does not rely on semantically embedding your code in a codebase or transforming it into a searchable structure. One of the things that makes Cloud Code effective is how it agentically reads through your code to take notes in file.cloud.md, figuring out autonomously what is going on in your codebase to then drive decision-making on how to advance your code.

That's right, and because it doesn't need to index the codebase, you can ensure the codebase stays local. We'll talk about some of the security ramifications of that. So let's get started, and I'll see you in the next video.
